案例模式 / Scenario Mode

這個案例示範道岔不到位如何從現場轉轍器一路影響聯鎖、告警維護與測試驗收流程。

事件解讀 / Event Trace
  1. 聯鎖命令轉轍器轉到 Reverse。
  2. 等待現場設備回報位置偵測。
  3. 此時尚未允許開放進路。

閉塞區間 / Block Sections

鐵路用區間佔用狀態控制列車間隔。當前方區間被列車佔用，後方號誌會限制或禁止進入。

目前佔用 / Occupied Block B

S2 顯示紅燈，因為列車正在佔用 Block B；S1 顯示黃燈，提醒前方下一區間受限。

聯鎖 / Interlocking

聯鎖會檢查進路、道岔、佔用狀態與衝突，只有安全條件成立時才允許開放號誌。

道岔與轉轍器 / Point Machine

轉轍器負責扳動道岔並回報位置。聯鎖必須確認道岔到位且鎖定，才會允許相關進路開放。

偵測回報 / Detection

列車偵測 / Train Detection

號誌系統需要先知道列車在哪裡。常見方式包含軌道電路與計軸器，兩者都用來判斷區間是否佔用。

判斷邏輯 / Detection Logic

軌道電路偵測到輪軸造成電氣分路，因此回報該區間已被佔用。

列車防護 / ATP & ATS

ATP 監控列車是否超過允許速度，必要時介入煞車；ATS 則偏向列車監督、營運管理與告警。

防護狀態 / Protection State

目前速度低於允許速度，ATP 持續監控但不介入。

調度控制 / CTC

CTC 讓控制中心集中監看多個車站、區間與進路狀態，協助調度員做營運決策。

調度重點 / Dispatch Focus

中心可看到各站進路與區間佔用，並維持列車依序運轉。

告警與維護 / Alarm & Maintenance

維護與測試時，控制中心需要快速辨識設備異常、通訊中斷、道岔不到位或偵測異常，並判斷影響範圍。

處置重點 / Maintenance Focus
  1. 確認道岔位置偵測是否一致。
  2. 暫停相關進路開放。
  3. 通知現場檢查轉轍器與鎖定機構。

測試與驗收 / Testing & Commissioning

號誌更新案需要透過分階段測試確認設計、現場 I/O、聯鎖邏輯、故障情境與試運轉結果，才能進入交付驗收。

驗收重點 / Acceptance Focus
  1. 確認每個輸入與輸出點位對應正確。
  2. 記錄測試結果、缺失與復測狀態。
  3. 通過後才能進入聯鎖邏輯測試。

平交道保護 / Level Crossing Protection

平交道系統會依列車接近偵測啟動警鈴、閃光號誌與柵欄，避免道路車輛進入列車路徑。

保護序列 / Protection Sequence

目前沒有列車接近，平交道維持開放，道路車輛可通行。

ETCS / ERTMS 入門

ERTMS 整合 ETCS、鐵路無線通訊與營運規則；ETCS 則把允許速度與行車權限帶到駕駛室。

資訊流 / Information Flow
  1. 路旁設備計算可授權範圍
  2. Balise / 應答器把資訊傳到車上
  3. 車上設備監控速度與行車許可
